## Description
OpenClaw for macOS turns a local AI agent runtime into a simple native Mac experience.

Instead of remembering terminal commands, ports, log locations, and setup steps, OpenClaw gives you a guided launcher for installing and controlling a local OpenClaw workspace.

Key features:
- Guided first-run setup for runtime installation and preferences.
- Menu bar control for starting and stopping the local OpenClaw server.
- One-click access to the local browser UI.
- Local configuration for data location, port, launch behavior, and provider access.
- Credentials stored through macOS Keychain where supported.
- Built-in diagnostics and logs export for troubleshooting.
- Official skills installer for OpenClaw workflows.
- Universal Mac support for Apple Silicon and Intel Macs.

OpenClaw is designed as a local-first developer tool. The local server binds to localhost, and cloud AI traffic is used only when the user configures an external provider.

This project is also an experiment in AI-native software creation: the implementation, documentation, scripts, packaging flow, and release materials were generated by AI agent workflows under human direction, review, and product judgment.

## Review notes draft
OpenClaw is a native macOS menu bar launcher for a local OpenClaw AI agent runtime.

The app is intended for developers and technical users who want a guided local setup instead of manual terminal commands.

Important review notes:
- The local server is bound to localhost.
- The app does not require an account to open the launcher.
- External AI provider usage is optional and configured by the user.
- Credentials configured through the app are stored through macOS Keychain where supported.
- The app is not a medical, financial, legal, or emergency tool.

Current upload status: build 17 / version 0.1.16 was uploaded to App Store Connect on 2026-05-20 and is processing. Nested Sparkle helpers and bundled Node were re-signed with App Store sandbox entitlements for upload analysis.

## Privacy draft
OpenClaw does not use advertising or third-party tracking.

The app may store local preferences, local runtime configuration, diagnostic logs, and provider credentials configured by the user. Provider credentials are stored locally through macOS Keychain where supported.

If the user configures an external AI provider, the content sent to that provider is governed by the provider selected by the user.
